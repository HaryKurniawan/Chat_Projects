Mitigations Applied

Fixes for VULN-01, VULN-02, VULN-03, VULN-04
1
VULN-01 FIX
Authenticate Socket.IO connections using the session cookie at handshake. Derive senderId from the verified JWT, never from client payload. 
io.use(async (socket, next) => {
  const token = socket.handshake.headers.cookie
    ?.match(/authToken=([^;]+)/)?.[1];
  if (!token) return next(new Error('Unauthorized'));
  const user = jwt.verify(token, process.env.JWT_SECRET);
  socket.data.user = user; // Store verified user
  next();
});

socket.on('sendMessage', async (data) => {
  const senderId = socket.data.user.id; // ← Server-verified!
  // ...
});
2
VULN-02 FIX
Remove role from register body. Strip any role field from the input entirely. 
// Remove the conditional role assignment:
await prisma.user.create({
  data: { name, email, password: hashedPassword },
  // role defaults to 'USER' in the schema
});
3
VULN-03/04 FIX
Sanitize with DOMPurify before rendering any user-generated content, especially if rich text or markdown is added later. 
import DOMPurify from 'dompurify';

// In AdminMainArea.tsx & ChatMainArea.tsx:
<p dangerouslySetInnerHTML={{
  __html: DOMPurify.sanitize(msg.content)
}} />

// DOMPurify strips all script/onerror payloads
4
VULN-07 FIX
Validate message length on the socket handler server-side. 
if (!content || typeof content !== 'string'
    || content.length > 2000) {
  socket.emit('error', 'Invalid message');
  return;
}
